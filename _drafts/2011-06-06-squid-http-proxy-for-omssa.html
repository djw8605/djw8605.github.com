Data in the Open Science Grid has always been more difficult to handle than computation.  But, there has always been three primary options:
  • SRM - Transfer files from reliable, but sometimes distant, large storage elements.
  • Staging - Pre-stage the data to sites in a global filesystem.
  • HTTP Caching - Using the site’s squid to cache HTTP requests to a server (probably at your institution)

Each solution has it’s own advantages.  SRM can handle large data stores, and bring down large amount of data.  Staging can distribute the load of pulling down data.  HTTP can move smaller requests than SRM, but will automatically stage the data.
